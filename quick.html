Three personas

Product Builder

Empowered with AI to ship. Owns Requirements, Mock-ups, Tech-Spec drafting, Dev + QA execution.

AR

Architect

Defines guardrails. Owns architectural integrity + sign-off. Reviewer, not implementer.

AM

Account Manager

Owns the client + the intent. Holds scope at Intent Check, drives UAT, owns Go-Live handover.

04

DevOps is the connective layer

Steps 01–05

Enable the personas

  • Delivery-posture catalogue (AM at Intent Check)
  • Requirement ↔ code traceability built in
  • Hosted preview URLs for mock-ups
  • ADR template + IaC catalogue (Architect signs against a shelf)

Steps 06–07

Build & QA, safely

  • AI-aware code guardrails (hallucination check, test-quality gate, ai-generated label)
  • Self-serve review environments < 15 min
  • UAT-tier with anonymised data + per-project status page

Steps 08–10

Release with evidence

  • Pre-Prod from same IaC modules · Vault · observability default-on
  • Kotwal scan as required gate · load test · evidence packet
  • Readiness portal · published Release Readiness Report

+ Beyond Go-Live

Operate & feedback

  • Mukhbir for runtime health · default observability stack
  • Weekly + monthly auto-reports · per-project status pages
  • Blameless incident review · quarterly portfolio review

DevOps recommendation: add an explicit 11th step for Operate & Feedback.

05

Project-start enablers — DevOps input

Stack & tooling

Starter repos with pre-commit, CI/CD, IaC and observability stubs already wired. Component-library catalogue. Mock-up tooling integrated with preview envs.

Sign-off & governance

Every sign-off is an artefact, not an email. Required-reviewer rules wire the Architect ↔ PB escalation. Tech-spec template co-designed with the Architect.

AI in the SDLC

Quality gates apply regardless of who or what wrote the code. DevOps prescribes the bar the output must clear, not which AI assistant a persona uses.

Rollout

Pilot criteria: exercises all 10 steps · not client-critical from day one · willing PM + Tech Lead. Sequence Tier S → M → L envs.
